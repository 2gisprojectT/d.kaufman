Техника Pairwise Testing
1. Провести анализ и оформить в письменном виде ответы на вопросы

• В чем суть метода (можно на небольшом примере)
	Метод тестирования построен, исходя из следующей идеи: большинство(но не все!) багов можно выявить проверяя
	один или сочетания 2-х параметров. Ошибки, которые стали результатом кобинаций 3-х и более параметров как правило
	значительно менее критичны.
	Суть метода в том, чтобы из всевозможных сочетаний пар параметров и их значений оставить только
	те, коорых достаточно для выполнения условия: Если взять любые 2 параметра из таблицы с тестами,
	то среди выбранных пар, будем иметь всемозможные сочетания значений выбранных параметров.
	Построить таблицу для системы можно с помощью ортогональных массивов и all-pairs алгоритма.
	Пример:
	Пусть у нас будет 3 взаимосвязанных параметра(P1, P2, P3) каждый из которых может принимать 3 различных
	значения(Z1, Z2, Z3). Т. е. каждый тест будет состоять из трех значений(по одному от каждого параметра).
	Если построим таблицу из всевозможных сочетаний, то будем иметь 3^3 = 27 записей.
	Если построим таблицу тестов согласно технике Pairwise Testing, то получим следующую таблицу из 15 строк:
	P1	P2	P3
				
	Z1	Z1	Z1	#############################
	Z1 	Z2 	Z2
	Z1	Z3	Z3
	Z2	Z1	Z1
	Z2	Z2	Z2		В этой части таблицы мы можем найти всевозможные сочетания значений
	Z2	Z3	Z3		пар параметров: (1 : 2), (1 : 3)
	Z3	Z1	Z1		а также значения пары (2 : 3) вида P2 : P3, где P2 == P3
	Z3	Z2	Z2
	Z3	Z3	Z3	#############################
		Z1	Z2		В этой части таблицы мы можем найти недостающие сочетания значений
		Z1	Z3		пары параметров (2 : 3) вида P2 : P3, где P2 != P3
		Z2	Z1		В первый столбец, на место пустых элементов мы можем вписать любые 
		Z2	Z3		из допустимых значения, таблица все равно будет содержать всевозможные
		Z3	Z1		значения всевозможных пар параметров
		Z3	Z2
		
		

• Для каких задач подходит

	Для задач, работа которых зависит от определенного количества параметров, имеющих определенное количество значений(Не слишком большие, так, чтобы количество тестов построенных по данной технике было выполнимым).
	Особо удобно использовать там, где нельзя выполнить полный перебор за обозримый промежуток времени.
	Техника не охватывает баги являющиеся комбинацией 3-х и более параметров, поэтому, данное тестирование следует проводить
	на поздних этапах разработки, когда другие тесты уже ненаходят баги, либо после тестирования основного функционала.
	Я считаю, что pairwise testing можно применить для любой задачи с подходящим числом параметров. При этом нужно помнить, что
	данная техника охватывает только часть ошибок, связанных со значениями одного параметра или какой либо пары параметров.
	Также следует помнить о следующем моменте: найденная данным способом тестирования ошибка, на самом деле может являтся результатом значений
	3-х и более параметров из которых нам известны только 2. Поэтому, размуно тестировать систему при параметрах, имеющих значение по умолчанию,
	кроме той пары, которую рассматриваем в данном случае.

• Для чего «не работает» или работает плохо
	Для задач, имеющих большое количество параметров и/или их значений, т.к. при данных условиях количество тестов будет невыполнимым
	Количество тестов в данном методе прямо пропорционально количеству тестов при полном переборе. Количество тестов при полном переборе
	= (количество значений)^(количество параметров), при равном количстве значений для всех параметров. Из выше перечисленного можно сделать
	вывод о том, что рост числа параметров вызовет более сильный рост числа тестов: как при полном переборе, так и при парном тестировании.
	Количество параметров более критично.
	
Soap opera testing
• В чем суть метода (можно на небольшом примере)
	Создать многошаговый тест, при выполнении которого перебрать как можно больше вариантов использования системы, с целью убедится, что при прохождении
	валидного сценрия не простейшим путем, система корректно выполнит свои функции.
	Создавая многошаговый тест по данной технике, на каждом шагу выбираем значения для параметров таким образом, чтобы они покрыли максимум допустимых вариантов, и потенциально проблемных сочетаний.
	При выполнении данного типа теста мы надеемся найти ошибки, вызванные, фактом последовательного выполнения функций системы при максимально
	большом тестовом покрытии на каждом шагу.
	
• Для каких задач подходит
	Для задач, имеющих много сценариев достижения цели, много ветвлений в алгоритме.
	
• Для чего «не работает» или работает плохо
	Для примитивных задач, у которых только один сценарий достижения цели.
